What Stage 1 does

Discovers candidate replication papers by querying OpenAlex with 23 keyword phrases. Results are deduplicated against candidates.csv using a persistent index (cache/candidates_index.txt) so re-runs never add duplicates. Only new DOIs are appended.

Papers matching any of the include phrases (and not matching any exclusion phrase) are written to candidates.csv. A cross-check against data/flora_entry_sheet.csv removes papers already in FLoRA.

CLI
python -m search.run_search

# Options
--from-year 2010      # filter by publication year
--to-year   2024
--rebuild-index       # rebuild cache/candidates_index.txt from scratch

What Stage 2 does

Removes false positives and classifies candidates into replication, reproduction, needs_review, or false_positive. Reads candidates.csv in 50 k-row chunks to avoid loading the full file into memory.

Rule-based classification runs first. Only rows the rules cannot decide are sent to the LLM. Results are streamed to filtered.csv via a resume index (cache/filtered_index.txt).

CLI
python -m filter.run_filter

# Options
--from-year 2010
--rebuild-index    # rebuild cache/filtered_index.txt
Decision logic
ConditionResult
Title/abstract matches exclusion pattern (DNA, code, …)false_positive
No include phrase foundfalse_positive
Include phrase + author-year citation patternreplication or reproduction
Include phrase, no citation — rule uncertain→ LLM
LLM uncertainneeds_review

LLM: OpenAI (FILTER_OPENAI_MODEL) primary; Gemini fallback. Responses cached by hash(title + abstract).

What Stage 3 does

For each replication paper: (1) classifies the match type, (2) finds the original target study, (3) codes the replication outcome. Results are streamed to extracted.csv one row at a time.

DOI verification runs automatically on every written row — it fetches CrossRef metadata for doi_o and corrects hallucinated DOIs before they hit the CSV.

CLI
python -m extract.run_extract

# Options
--extracted-test   # write to extracted-test.csv (sandbox)
--resume           # skip already-processed DOIs
--no-llm           # rule-based only (faster, less accurate)
--doi 10.x/y       # process a single DOI

# DOI audit
python -m extract.audit_dois          # dry-run
python -m extract.audit_dois --apply  # write corrections

What Extract-Test is

A sandbox for running and reviewing Stage 3 pipelines before committing results to extracted.csv. Writes to extracted-test.csv instead, and automatically skips DOIs already resolved in extracted.csv.

Use it to test new prompts, new models, or multi-original rows without risking the production file. Promote individual rows or everything when satisfied.

CLI
# Run into sandbox
python -m extract.run_extract --extracted-test
python -m extract.run_extract --extracted-test --resume

# Promote to production
python -m extract.promote_test --all           # promote all
python -m extract.promote_test --doi 10.x/y   # promote one
python -m extract.promote_test --all --dry-run # preview

What Stage 4 does

Human validators review each extracted row — checking the original DOI, replication type, and outcome. Votes are stored in Supabase (PostgreSQL). Two independent validators vote per row; disagreements are flagged for adjudication.

Configuration
SUPABASE_URL=https://xxx.supabase.co
SUPABASE_SERVICE_KEY=eyJhbGc...

Add these to your .env file. Without them, this tab shows "not configured".

What this section covers

Gap analysis comparing what the current pipeline has extracted against the FLoRA entry sheet (107 known replications). Use it to find missed papers, filter misclassifications, and plan re-extraction.

All scripts are in analysis/. Data files live in analysis/ and are not committed to git.

CLI
python -m analysis.run_overlap_analysis  # gap analysis
python -m analysis.rule_analysis         # filter rule audit
python -m analysis.apa_resolver          # resolve APA citations
Analysis scripts
  • run_overlap_analysis.py Gap analysis — compares extracted.csv against FLoRA entry sheet. Writes gap CSVs and gap_summary.md.
  • rule_analysis.py Audits filter rules: contribution per rule, false-positive rate, improvement opportunities.
  • apa_resolver.py Resolves APA-format citations in extracted rows to structured DOIs via CrossRef.
Output files
  • gap_analysis_doi_matched.csvDOI-matched gap list
  • gap_analysis_url_matched.csvURL-matched gaps
  • gap_vs_extracted_comparison.csvSide-by-side comparison
  • rule_improvement_opportunities.csvActionable filter improvements
  • gap_summary.mdHuman-readable summary
